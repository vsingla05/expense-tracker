     Plan: Profile Page Design (Step 4)

     Context

     The spec calls for building a static UI-only profile page with hardcoded
     data. This establishes the complete layout before wiring up real database
     queries in Step 5. The profile page needs to display:
     1. User info card (avatar initials, name, email, member-since)
     2. Summary stats row (total spent, transaction count, top category)
     3. Transaction history table (hardcoded rows)
     4. Category breakdown section

     This is a frontend-only step - all data will be hardcoded Python
     dicts/lists in app.py, no database queries yet.

     Critical Files

     To Create:

     - templates/profile.html — new template extending base.html

     To Modify:

     - app.py — replace the /profile stub with a real view that passes hardcoded
      context

     Design Approach

     Following the frontend-design/SKILL.md guidelines:

     Visual Style

     - Use existing CSS variables from style.css (no hardcoded hex values)
     - Card-based layout with white surfaces, subtle borders (var(--border)),
     soft shadows
     - 8px spacing grid, consistent radius (var(--radius-md) for cards)
     - Lucide icons for visual polish (loaded via CDN in base.html)

     Page Structure

     1. User Info Card — left sidebar style with avatar (initials), name, email,
      join date
     2. Summary Stats — 3 cards in a row (total spent, transactions, top
     category)
     3. Transaction Table — recent expenses with date, description, category
     badge, amount
     4. Category Breakdown — progress-bar style list showing per-category totals

     Hardcoded Data (in app.py)

     user = {"name": "Demo User", "email": "demo@spendly.com", "member_since":
     "April 2026"}
     stats = {"total_spent": 453.89, "transaction_count": 8, "top_category":
     "Shopping"}
     transactions = [
         {"date": "2026-04-15", "description": "Grocery shopping", "category":
     "Food", "amount": 45.50},
         {"date": "2026-04-12", "description": "Restaurant dinner", "category":
     "Food", "amount": 32.50},
         {"date": "2026-04-11", "description": "New shoes", "category":
     "Shopping", "amount": 89.99},
         # ... more rows
     ]
     categories = {"Food": 78.00, "Shopping": 89.99, "Transport": 25.00,
     "Bills": 120.00}

     Implementation Steps

     1. Create templates/profile.html
       - Extend base.html
       - Four sections: user card, stats row, transaction table, category
     breakdown
       - Use Jinja variables passed from view
       - Category badges use CSS classes (.badge-food, .badge-shopping, etc.)
     2. Update app.py /profile route
       - Keep @login_required decorator
       - Replace stub return with render_template("profile.html", ...)
       - Pass hardcoded dicts for user, stats, transactions, categories
     3. Add CSS to style.css (or new profile.css)
       - Profile grid layout
       - Stats card styling
       - Transaction table styles
       - Category badge colors (use CSS variables, not hex)
       - Progress bar components
     4. Add Lucide icons (optional polish)
       - Wallet, trending-up, calendar icons
       - Call lucide.createIcons() after DOM ready

     Verification

     1. Run python app.py and navigate to /profile while logged in
     2. Verify unauthenticated access redirects to /login
     3. Check all four sections render with hardcoded data
     4. Confirm no hex colors in template — only CSS variables
     5. Verify navbar shows logged-in state with username + logout link
     6. Test responsive behavior at narrow widths
